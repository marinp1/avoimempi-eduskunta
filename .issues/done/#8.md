# Improve Session Tab representation

* Show contents of each parliamentary session (agenda).
* Consists of multiple _sections_ which each represent a discussion held within the session. Each section has a theme, a speaker and most likely some sort of speeches.

Consult domain expert how the view should be structures.

## Completed Implementation

### Backend Changes
1. Created SQL query: `packages/server/database/queries/SECTION_VOTINGS.sql`
2. Added query export in `packages/server/database/queries.ts`
3. Added `fetchSectionVotings` method to `DatabaseConnection` class
4. Added API route: `GET /api/sections/:sectionKey/votings`

### Frontend Changes
1. Added votings display to Sessions page (`packages/client/pages/Sessions/index.tsx`)
2. Votings are fetched on-demand when section is expanded (parallel with speeches)
3. Visual display includes:
   - Vote number badge
   - Vote title
   - Result badge (Hyväksytty/Hylätty) with color coding (green/red)
   - Vote counts: Kyllä, Ei, Tyhjä, Poissa, Yhteensä
   - Border color indicates result (green for passed, red for failed)
   - HowToVoteIcon for visual consistency

### Section Metadata Improvements
1. Show `processing_title` if different from `title`
2. Show `resolution` text if available
3. Display `identifier` for reference

### Result
Sessions page now properly displays:
- Parliamentary sessions with their agendas
- Expandable sections (kohdat) within each session
- Votings for each section (when available)
- Speeches for each section (when available)
- Empty state message when neither votings nor speeches exist
