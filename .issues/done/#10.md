# Improve application performance

Since the database can be considered static and will be updated on a schedule (daily), the queries could be _heavily_ cached.

Implement a cache solution to store query results with a possibilty to invalidate them on demand (e.g. deleting the cache entry manually or similar). As the database has some "common" queries that are ALWAYS executed (i.e. sanity checks, day views etc. that are loaded on load), those could be executed and cached BEFORE the server starts up to make sure that initial loads are OK.
