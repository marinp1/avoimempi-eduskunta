# Improve document preview formatting

Current type-specific extraction is functionally correct, but preview readability can be improved.

Follow-up scope:

1. Add lightweight formatting support for extracted document previews in `/istunnot`:
   - section headings
   - paragraph spacing
   - list rendering for participant/person/group lines
2. Keep current metadata filtering behavior (`PhraseID`, `PhrasesetID`, GUID/control values must remain excluded).
3. Ensure formatting stays deterministic across document types and does not break fallback plain-text rendering.
4. Add tests/snapshots for formatted output per supported type.
